# Production Send — Material Between Steps

## What it does

Records material moving from one step to the next. Partial sends allowed.

## Endpoints

### Send material to next step

`POST /plan-items/{id}/sends`

**Request**

```json
{
  "from_step_id": 1,
  "to_step_id": 2,
  "from_brigada_id": 5,
  "to_brigada_id": 8,
  "kg_sent": 200.0,
  "meters_sent": 1500.0,
  "notes": "Birinchi partiya"
}
```

**Response (201)**

```json
{
  "id": 1,
  "plan_item_id": 10,
  "from_step_id": 1,
  "to_step_id": 2,
  "from_brigada_id": 5,
  "to_brigada_id": 8,
  "kg_sent": 200.0,
  "meters_sent": 1500.0,
  "sent_by": 1,
  "notes": "Birinchi partiya",
  "created_at": "2026-03-01T14:00:00"
}
```

### What happens on send

1. `ProductionSend` record created
2. `to_step.kg_received` updated: `+= kg_sent` (summed from all sends)
3. If `to_step.status == pending`:
   - `to_step.status` → `in_progress`
   - `to_step.started_at` = now
   - `to_step.brigada_id` = `to_brigada_id`
   - `to_step.machine_id` = assigned machine

### Partial sends

One step can send multiple times:

```
ProductionSend #1: pechat → sushka, 200kg
ProductionSend #2: pechat → sushka, 280kg

Sushka total kg_received = 200 + 280 = 480kg
```

### Validation rules

- `from_step` must belong to same `plan_item_id`
- `to_step` must belong to same `plan_item_id`
- `to_step.step_number` must be `from_step.step_number + 1` (sequential only)
- `from_step.status` must be `in_progress` or `completed`
- `kg_sent` must be > 0
- `kg_sent` cannot exceed `from_step.kg_produced - already_sent` (can't send more than produced)

### Flow example (full_cycle)

```
Send 1: reska(1)     → pechat(2)       480kg
Send 2: pechat(2)    → sushka(3)       200kg  (partial)
Send 3: pechat(2)    → sushka(3)       270kg  (rest)
Send 4: sushka(3)    → laminatsiya(4)  470kg
Send 5: laminatsiya(4) → sushka(5)     540kg
Send 6: sushka(5)    → reska(6)        540kg
Send 7: reska(6)     → tayyor(7)       460kg  ← triggers tayyor ombor logic
```
