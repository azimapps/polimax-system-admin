# Tayyor — Final Step & Ombor Entry

## What happens when material reaches TAYYOR

When a `ProductionSend` targets the last step (tayyor), the system:

1. Creates or updates an `OmborItem` in `tayyor_angren` (always angren)
2. Creates an `OmborTransaction` (kirim) for history
3. If plan item's all steps are completed → plan item `status = finished`

## Flow

### First send to tayyor

```
ProductionSend: reska → tayyor, 200kg
```

System auto-creates:

```
OmborItem (tayyor_angren):
  name: from order name
  ombor_type: tayyor_angren
  quantity: 200
  product_type: from order
  date: now

OmborTransaction:
  ombor_item_id: (new item)
  transaction_type: kirim
  quantity_kg: 200
  plan_item_id: 10
```

### Subsequent sends to tayyor (same order)

```
ProductionSend: reska → tayyor, 150kg (second batch)
```

System updates existing OmborItem:

```
OmborItem:
  quantity: 200 + 150 = 350  (summed up)

OmborTransaction:
  ombor_item_id: (same item)
  transaction_type: kirim
  quantity_kg: 150
  plan_item_id: 10
```

### Multiple partial sends — full history

```
Send #1: 200kg  →  OmborTransaction (kirim 200kg, 10:00)
Send #2: 150kg  →  OmborTransaction (kirim 150kg, 14:00)
Send #3: 110kg  →  OmborTransaction (kirim 110kg, 16:00)

OmborItem total: 460kg
History: 3 separate transactions, full trace
```

## Rules

- Always `tayyor_angren` ombor type (never toshkent)
- Same order = same OmborItem (sends add up)
- Each send = separate OmborTransaction for full history
- When ALL steps are `completed` → PlanItem.status = `finished`
