# Steps — Production Step Tracking

## Step Types

| step_type | Description | Materials from ombor? | kg loss? |
|-----------|-------------|----------------------|----------|
| `reska` | Cutting plyonka to size | Plyonka (if first step) | Yes (waste, ostatok) |
| `pechat` | Printing design on plyonka | Kraska | Yes (waste, ostatok) |
| `sushka` | Drying — just countdown | No | No (kg_in = kg_out) |
| `laminatsiya` | Laminating layers | Kley, plyonka (extra layer) | kg_out > kg_in (layers added!) |
| `tayyor` | Finished — goes to ombor | No | No |

## Step Statuses

```
pending → in_progress → completed
```

### Get single step

`GET /plan-items/{id}/steps/{step_id}`

**Response (200)** — same as single item from list above.

### Update step (start / update progress / complete)

`PUT /plan-items/{id}/steps/{step_id}`

#### Start a step

```json
{
  "status": "in_progress",
  "brigada_id": 8,
  "machine_id": 6
}
```

#### Update production progress

```json
{
  "kg_produced": 200.0,
  "kg_waste": 5.0,
  "kg_ostatok": 15.0,
  "meters_produced": 1200.0
}
```

#### Complete a step

```json
{
  "status": "completed",
  "kg_produced": 470.0,
  "kg_waste": 8.0,
  "kg_ostatok": 2.0,
  "meters_produced": 3500.0
}
```

**Response (200)** — updated step object.

### Validation rules

- Cannot start step if previous step is still `pending`
- Cannot complete step without `kg_produced`
- Sushka: `kg_produced` must equal `kg_received`
- Sushka: `kg_waste` and `kg_ostatok` must be 0 or null
- `brigada_id` and `machine_id` required when starting a step

### Sushka special rules

- No materials from ombor
- No kg loss: `kg_produced = kg_received`
- Brigada = NEXT step's brigada (not sender's)
- Just a countdown/timer
- `started_at` to `completed_at` = drying time

### Balance check (non-sushka steps)

```
kg_received = kg_produced + kg_waste + kg_ostatok
```

Exception: laminatsiya — `kg_produced` can exceed `kg_received` because layers are added from ombor materials.
