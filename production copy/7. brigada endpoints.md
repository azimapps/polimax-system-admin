# Brigada Endpoints — Worker-Facing API

All endpoints require brigada leader authentication.

Base path: `/material-usage`

---

## GET /material-usage/my-steps

Returns all pipeline steps assigned to the caller's brigada. This is how a brigada discovers what work is waiting for them (sushka, pechat, etc.) without needing to know the plan_item_id.

**Query Parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `status` | string | Filter: `pending`, `in_progress`, or `completed`. Default: returns `pending` + `in_progress` |

**Examples**

```
GET /material-usage/my-steps
GET /material-usage/my-steps?status=in_progress
```

### Use case: Sushka brigada

Sushka brigada calls `GET /my-steps?status=in_progress` and sees all sushka steps waiting for them. Each item shows `kg_received` (how much material arrived) and `order_title` (which order). Then they call `POST /material-usage/production-log` with `plan_item_id` + `send` to complete and forward.

## GET /material-usage/plan-item/{plan_item_id}/steps

Returns all pipeline steps for a plan item.

**Response (200)** — list of step objects (see `2. steps.md`).

---

## GET /material-usage/plan-item/{plan_item_id}/sends

Returns all send records for a plan item.

**Response (200)** — list of send objects (see `3. production-send.md`).

## POST /material-usage/production-log

The main production endpoint. Creates production logs for all brigada members + optional material usage + optional send to next step.

See `6. production-log.md` and `3. production-send.md` for details.

**Request**

```json
{
  "plan_item_id": 10,
  "meters_produced": 3500.0,
  "kg_produced": 470.0,
  "work_type": "tayyor_mahsulotlar_reskasi",
  "percentage": 100.0,
  "notes": "Pechat tugadi",
  "materials": [
    {
      "ombor_transaction_id": 56,
      "used_amount": 28.0,
      "remainder_destination": "machine_warehouse",
      "notes": "Kraska"
    }
  ],
  "send": {
    "to_brigada_id": 8,
    "kg_sent": 470.0,
    "meters_sent": 3500.0,
    "kg_waste": 5.0,
    "kg_ostatok": 5.0,
    "notes": "Sushkaga yuborildi"
  }
}
```

**Response (201)** — list of ProductionLog objects (one per brigada member).
