# Production Log — Per-Worker Tracking

## What it does

Records individual worker production output within a step. Used for performance tracking and salary calculation.

## Model

```
ProductionLog:
  plan_item_id    → which order
  brigada_id      → which brigada
  machine_id      → which stanok
  worker_id       → which staff member (individual worker)
  meters_produced → how many meters this worker produced
  kg_produced     → how many kg this worker produced
  work_type       → type of work done
  percentage      → worker's contribution percentage
  notes           → optional notes
```

## Work Types

| Value | Description |
|-------|-------------|
| `tayyor_mahsulotlar_reskasi` | Cutting finished products |
| `tayyor_mahsulot_peremotkasi` | Rewinding finished products |
| `plyonka_peremotkasi` | Rewinding plyonka |
| `reska_3_5_sm` | Cutting 3-5 cm |
| `asobiy_tarif` | Special tariff work |

## Endpoints

### Create production log

`POST /material-usage/production-logs`

**Request**

```json
{
  "plan_item_id": 10,
  "brigada_id": 5,
  "machine_id": 3,
  "worker_id": 22,
  "meters_produced": 500.0,
  "kg_produced": 80.0,
  "work_type": "tayyor_mahsulotlar_reskasi",
  "percentage": 25.0,
  "notes": "Birinchi smena"
}
```

**Response (201)**

```json
{
  "id": 1,
  "plan_item_id": 10,
  "brigada_id": 5,
  "machine_id": 3,
  "worker_id": 22,
  "meters_produced": 500.0,
  "kg_produced": 80.0,
  "work_type": "tayyor_mahsulotlar_reskasi",
  "percentage": 25.0,
  "notes": "Birinchi smena",
  "created_at": "2026-03-01T18:00:00"
}
```

### List production logs for a plan item

`GET /material-usage/production-logs?plan_item_id=10`

**Response (200)** — list of production log objects.

### List production logs for a worker

`GET /material-usage/production-logs?worker_id=22`

**Response (200)** — all logs for that worker across orders.

## Relationship to steps

Production logs are per plan item, per worker. They exist alongside the step system but are not directly linked to a specific step — they track individual worker output for salary purposes.

Multiple workers in one brigada can log their individual contributions:

```
Brigada 5 on Step 2 (pechat), total kg_produced = 470:
  Worker 22: 120kg, 25%
  Worker 23: 180kg, 38%
  Worker 24: 170kg, 37%
```
