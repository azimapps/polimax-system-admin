# AI Sheet Parsing Integration Guide

This document describes the AI-powered Excel sheet parsing integration implemented in the Polimax System Admin.

## Overview

The AI Sheet Parsing feature allows warehouse managers to import large datasets from Excel (.xlsx) files. Instead of a rigid mapping, an AI (Large Language Model) is used to:
1.  **Analyze** the uploaded file structure.
2.  **Ask clarifying questions** to the user for ambiguous data (e.g., matching column headers to system fields, resolving categorical values).
3.  **Validate and Normalize** the data before final import.

## Technical Architecture

The integration follows a multi-step session-based interaction between the Frontend and Backend.

### 1. API Endpoints (`omborApi`)

All endpoints are scoped to the specific warehouse type (e.g., `plyonka`).

-   **`POST /ombor/:type/parse-sheet`**: Uploads the Excel file and starts an AI parsing session. Returns a `session_id` and the initial status (either `questions` or `complete`).
-   **`POST /ombor/:type/parse-sheet/:sessionId/answer`**: Submits answers to the AI's clarifying questions. Returns the updated status.
-   **`GET /ombor/:type/parse-sheet/:sessionId/download`**: Downloads a cleaned and normalized version of the uploaded Excel file.

### 2. UI Components

#### `OmborImportAiDialog`
The main interactive component (`src/sections/ombor/ombor-import-ai-dialog.tsx`) that manages the parsing lifecycle:
-   **Idle State**: File upload zone.
-   **Uploading State**: Shows a "Processing..." message while the AI analyzes the data.
-   **Questions State**: Dynamically renders a form with selectors and inputs based on questions generated by the AI.
-   **Complete State**: Displays a summary of the parsing results (Total Rows, Valid Rows, Errors) and a preview table.

### 3. Data Structure (Plyonka Example)

The AI is specifically tuned to extract the following fields for the "plyonka" page:
-   `name`: Item name.
-   `plyonka_category`: (BOPP, CPP, PE, PET).
-   `thickness`: Numerical value.
-   `width`: Numerical value.
-   `total_kg`: Total weight.
-   `price_per_kg`: Unit price (normalized to USD).

## Workflow

1.  **Select File**: User drops a `.xlsx` file.
2.  **Initial Analysis**: AI identifies the main columns. If column headers are unclear (e.g., "Eni" instead of "Width"), it marks them for clarification.
3.  **Interaction**: User answers questions like:
    *   "Which column represents the Thickness?"
    *   "Does 'Kley' refer to a specific category?"
4.  **Verification**: AI shows the preview of what it parsed.
5.  **Finalization**: User clicks "Save to System", and the component iterates through the parsed items, creating them via the standard `omborApi.createOmborItem`.

## Error Handling

-   **Invalid Format**: If the file is not a valid Excel file, the initial parse request will fail.
-   **Ambiguous Data**: The AI will proactively ask questions rather than making incorrect assumptions.
-   **Persistence Errors**: If individual items fail to create, they are logged, and the process continues.

## Maintenance

To update the AI parsing logic or field mappings, changes should be made to the Backend AI prompt orchestration. The Frontend is generic and handles dynamic questions returned by the API.
